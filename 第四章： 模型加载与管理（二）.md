# 第4章 模型加载与管理（二）

模型加载与管理是 vLLM 推理的基础环节，直接决定模型部署的兼容性、稳定性与灵活性。vLLM  支持多种主流模型格式，适配从 HuggingFace Hub 远程加载、本地路径加载等多种场景，同时支持多模型服务部署与动态切换，满足不同生产需求。本章严格遵循规范，按“格式支持→加载方法→管理技巧”的逻辑讲解，每个模块均配套实操示例，兼顾教学易懂性与生产实用性。

## 4.1 支持的模型格式：PyTorch 原生格式、Safetensors、GGUF

vLLM   兼容三大主流模型格式，覆盖不同使用场景（远程部署、本地离线、轻量推理），无需额外格式转换工具，直接加载即可使用，官方明确支持的格式及适配场景如下，贴合教学重点分类讲解。

### 4.1.1 核心支持格式

1. PyTorch 原生格式最传统，基于 Python 的 pickle 序列化。但存在安全风险（可能执行恶意代码），且加载速度较慢。.bin 或 .pt早期模型、本地快速调试

   

2. Safetensors 格式（安全优先）定义：一种安全的模型权重格式，以 .safetensors 为后缀，相比 HuggingFace 原生的 .bin 格式，可避免恶意代码注入，加载速度更快，且支持内存映射，降低显存占用。核心优势：安全、高效，加载时无需担心恶意代码，适合生产环境中对安全性要求较高的场景。适用场景：生产部署、多用户共享模型环境，尤其适合企业级推理场景。注意事项：vLLM 自动识别 Safetensors 格式，无需额外参数，当模型目录中同时存在 .bin 和 .safetensors 文件时，优先加载 .safetensors 文件。这是强力推荐的格式

   

3. GGUF 格式（轻量推理）官方定义：由 llama.cpp 推出的轻量级模型格式，以 .gguf 为后缀，支持模型量化（如 4-bit、8-bit），体积小、加载速度快，适配低配置 GPU 或 CPU 推理场景。它将模型权重、配置等打包成一个文件，非常适合在本地或资源受限的设备上运行。核心优势：轻量、高效，量化后模型体积大幅缩小，可在低显存设备上运行，加载速度比 HuggingFace 格式快 20%~50%。适用场景：CPU 推理、低显存 GPU 推理、嵌入式设备部署，适合轻量级推理需求。注意事项：加载 GGUF 格式模型时，需指定模型文件名（如 --model model.gguf），且仅支持部分模型（如 LLaMA 系列、Qwen 系列），具体兼容列表可参考 vLLM 官方文档。

### 4.1.2 格式兼容性总结

- 优先选择：Safetensors 格式（生产安全优先）；

- 轻量需求：GGUF 格式（量化后体积小，适配低配置设备）；

- 禁忌：避免使用自定义非标准格式，会导致模型加载失败，若需转换格式，可使用 HuggingFace Transformers、safetensors 等官方工具。

## 4.2 从 HuggingFace Hub 加载模型

HuggingFace Hub 是目前最主流的大模型开源仓库，vLLM v0.7.0 支持直接从 Hub 加载模型，无需手动下载，只需指定模型名称即可自动完成下载与加载，操作简洁，适配教学与快速演示场景。

### 4.2.1 核心加载逻辑

加载流程：指定 HuggingFace Hub 模型名称 → vLLM 自动检查本地缓存 → 无缓存则从 Hub 下载（默认保存至 ~/.cache/huggingface/hub）→ 加载模型至 GPU/CPU → 完成部署。

核心参数：`--model <hub-model-name>`，模型名称需与 HuggingFace Hub 上的模型名称完全一致（区分大小写）。

### 4.2.2 实操示例（官方推荐）

1. 加载 Safetensors 格式模型
   若模型 Hub 仓库中包含 Safetensors 格式文件，vLLM 自动优先加载，无需额外参数：

     ```
     (base) root@bd3c17e866d2:/data# vllm serve Qwen/Qwen3-0.6B --host 0.0.0.0 --port 8000 --trust-remote-code
     `说明：--trust-remote-code 用于加载模型的自定义代码（部分模型需开启），--dtype float16 指定使用半精度加载，降低显存占用。
     ```

     ![image-20260303111122966](http://www.410166399.xyz/image-20260303111122966.png)

3. 指定缓存路径（自定义下载位置）**通过 `--model-cache-dir <path>` 参数指定模型下载缓存路径，避免默认路径占用系统盘空间：

     ```
     (base) root@bd3c17e866d2:/data# vllm serve Qwen/Qwen3-0.6B --host 0.0.0.0 --port 8000 --trust-remote-code --model-cache-dir  /data
     ```

     

### 4.2.3 加载注意事项（官方强调）

- 网络要求：从 Hub 加载模型需联网，且网络稳定，大模型（如 7B 及以上）建议使用高速网络，避免下载中断；

- 权限问题：若加载私有模型，需先通过 `huggingface-cli login` 命令登录 HuggingFace 账号，获取访问权限；

- 版本兼容：确保 Hub 模型版本与 vLLM 兼容，建议选择模型的稳定版（如 v1.5、v2），避免使用开发版。

## 4.3 本地模型加载与路径配置

针对离线环境、自定义模型或频繁加载的场景，vLLM 支持从本地路径加载模型，只需指定本地模型目录路径即可，无需联网，加载速度更快，适配离线教学与生产部署场景。

### 4.3.1 本地模型目录规范（官方要求）

无论哪种模型格式，本地模型目录需满足以下规范，否则会导致加载失败：

1. 目录结构清晰，包含模型核心文件（如 config.json、权重文件、tokenizer 文件）；

2. 权重文件名称符合官方规范（pytorch原生格式：pytorch_model.bin；Safetensors 格式：model.safetensors；GGUF 格式：*.gguf）；

3. 路径无中文、特殊字符（如空格、括号），建议使用绝对路径，避免路径错误。

### 4.3.2 实操示例（不同格式本地加载）

1. 本地 Safetensors 格式模型加载假设本地模型目录为 /data/models/Qwen-7B-Chat-safetensors，目录内包含 model.safetensors，加载命令：

     ```
     (base) root@bd3c17e866d2:/data# vllm serve /mnt/moark-models/Qwen3-0.6B/ --host 0.0.0.0 --port 8000
     ```
     

2. 本地 GGUF 格式模型加载假设本地 GGUF 模型文件为 /data/models/llama-2-7b-chat.Q4_K_M.gguf，加载命令（需指定完整文件名）：

     ```
     vllm serve /mnt/mork-models/Qwen3.5-4B-GGUF\
     --trust-remote-code \
     --dtype float16 \
     --port 8000 \
     --host 0.0.0.0
     ```

离线批量推理是 VLLM 核心应用场景之一，适用于无需实时响应、需处理大量 prompt 推理任务的场景（如数据标注、模型微调数据生成、批量内容生成等）

## 2.1 核心概念

VLLM 离线批量推理，核心是通过 LLM 引擎加载模型后，一次性输入批量 prompt，批量执行推理计算并输出结果，相比单条推理，能最大化利用 GPU 资源，提升推理吞吐量（单位时间内处理的 prompt 数量），降低推理成本。

官方核心定义梳理，精准理解关键术语：

- 离线推理：非实时推理，无需部署在线服务，直接通过 Python 脚本执行，适合批量处理任务；

- 批量推理：一次性输入多个 prompt（列表形式），模型批量处理，共享 GPU 资源，提升效率；

- LLM 引擎：VLLM 推理的核心组件，负责模型加载、显存优化、推理计算，是离线推理的基础；

- 采样参数：控制推理结果的生成逻辑（如随机性、长度限制），是离线推理的核心配置参数；

- KV 缓存：VLLM 核心优化技术，缓存推理过程中的键值对，减少重复计算，提升批量推理速度。

官方关键说明：离线批量推理的核心优势是「高吞吐量、低资源占用」，适合处理千级、万级甚至更高量级的 prompt 推理任务，Python API 调用是官方推荐方式，支持自定义结果解析、异常处理，适配复杂离线场景。

## 2.2 Python API 调用

VLLM 官方优先推荐使用 Python API 进行离线批量推理，支持自定义采样参数、批量输入、结果解析、异常处理，适配复杂的离线推理场景（如批量生成、结果保存到文件等）。本节将先重点说明离线推理核心的 3 个关键类，再讲解完整实操流程，所有内容严格遵循官方标准。

### 2.2.1 离线推理 3 个关键类详细说明

VLLM 离线批量推理的核心依赖 3 个关键类，分别是 LLM（模型引擎类）、SamplingParams（采样参数类）、LLMOutput（推理结果类），三者协同完成从模型加载、参数配置到推理输出的全流程，以下是官方标准说明：

#### 1. LLM 类（核心模型引擎类）

官方定义：LLM 类是 VLLM 推理的核心引擎类，负责加载目标模型、初始化 GPU 资源、管理推理任务调度，是离线批量推理的基础，所有推理操作均依赖该类的实例。

核心作用：

- 加载模型：支持加载 Hugging Face 在线模型（如 Qwen/Qwen2.5-1.5B-Instruct）或本地模型文件，自动适配模型结构；

- 资源分配：根据配置自动分配 GPU 显存、CPU 资源，支持张量并行（多 GPU 场景）；

- 推理调度：接收批量 prompt，调度 GPU 执行推理计算，调用 KV 缓存优化推理速度；

- 核心方法：generate() 方法，是执行批量推理的核心方法，接收 prompt 列表和采样参数，返回推理结果。

官方注意：LLM 类初始化时会加载模型到 GPU，初始化一次即可重复执行推理，无需多次加载（避免显存浪费）。

#### 2. SamplingParams 类（采样参数配置类）

官方定义：SamplingParams 类用于配置推理结果的生成规则，控制生成内容的随机性、长度、多样性等，所有采样参数均通过该类实例传入 LLM 类的 generate() 方法。

核心作用：

- 控制随机性：通过 temperature 参数调节生成内容的随机性，适配不同推理场景；

- 限制长度：通过 max_tokens 参数限制生成内容的最大长度，避免生成冗余内容；

- 优化多样性：通过 top_p、presence_penalty 等参数优化生成内容，减少重复、提升丰富度；

- 终止控制：通过 stop 参数设置停止符，确保生成内容符合预期格式。

官方注意：该类参数均有官方默认值，可根据实际需求调整，无需手动配置所有参数。

#### 3. LLMOutput 类（推理结果返回类）

官方定义：LLMOutput 类是 generate() 方法的返回值类型，用于封装批量推理的所有结果，每个 prompt 对应一个 LLMOutput 实例，存储输入 prompt、生成结果、推理耗时等信息。

核心属性（常用，官方推荐）：

- prompt：输入的单条 prompt 字符串，对应批量推理中的一条输入；

- outputs：生成结果列表（单条 prompt 对应一个结果，故列表长度为 1）；

- outputs[0].text：单条 prompt 对应的推理生成文本，是核心结果；

- outputs[0].token_ids：生成文本对应的 token 编号，用于进阶开发；

- time_elapsed：该条 prompt 的推理耗时（秒），用于性能分析。

官方注意：generate() 方法返回的是 LLMOutput 实例列表，列表长度与批量 prompt 数量一致，需循环解析每个实例获取结果。

### 2.2.2 实操步骤

基于上述 3 个关键类，VLLM 离线批量推理（Python API）核心分为 5 步，官方标准流程，无多余步骤，确保高效落地：

1. 环境准备：确认已完成 VLLM 基础环境安装（参考第一章 1.1 节），确保 CUDA 版本 ≥ 11.8，GPU 显存充足；

2. 导入依赖：导入 VLLM 核心模块（LLM、SamplingParams），无需额外导入其他依赖；

3. 配置采样参数：通过 SamplingParams 类实例配置采样参数，根据需求调整；

4. 初始化 LLM 引擎：通过 LLM 类实例加载目标模型，配置模型精度、显存优化等参数；

5. 执行批量推理：调用 LLM 实例的 generate() 方法，输入批量 prompt，解析 LLMOutput 列表获取结果。

### 2.2.3 完整代码示例

以下代码为 VLLM 官方提供的离线批量推理完整示例，适配 Qwen2.5-1.5B-Instruct 模型（官方推荐入门模型），整合 3 个关键类的使用，支持批量输入、结果解析、保存到文本文件，可直接复制运行，仅需修改 prompt 列表和模型路径。

```python

from vllm import LLM, SamplingParams

if __name__ == '__main__':
    # 1. Configure sampling parameters
    sampling_params = SamplingParams(
        temperature=0.7,          # randomness; 0 = deterministic, 1 = maximum randomness
        max_tokens=150,           # maximum number of tokens to generate
        top_p=0.95,               # nucleus sampling; controls diversity
        stop=["\n", "###"],       # stop sequences; generation stops when these are encountered
        presence_penalty=0.1,     # penalty for repeating content; reduces repetition
        frequency_penalty=0.1     # frequency penalty; reduces occurrence of high-frequency words
    )

    # 2. Initialize the LLM engine (load the model)
    llm = LLM(
        model="Qwen/Qwen2.5-1.5B-Instruct",  # Hugging Face model ID or local path
        trust_remote_code=True,               # required for custom models like Qwen
        dtype="auto",                          # inference precision; 'auto' auto‑selects
        max_model_len=4096,                    # maximum sequence length for the model
        load_format="auto",                    # model loading format; 'auto' picks optimal method
    )

    # 3. Prepare a batch of prompts
    batch_prompts = [
        "Explain in detail how offline batch inference works in VLLM.",
        "Compare the offline inference performance of VLLM and Hugging Face Transformers.",
        "Summarize the core steps of batch inference using the VLLM Python API.",
        "Generate three short phrases about the development of artificial intelligence, each no longer than 20 words.",
        "What is KV cache, and what role does it play in VLLM inference?"
    ]

    # 4. Perform batch inference
    outputs = llm.generate(batch_prompts, sampling_params)  # returns a list of LLMOutput objects

    # 5. Parse and display results (using ASCII brackets to avoid encoding issues)
    results = []
    for output in outputs:
        prompt = output.prompt
        generated_text = output.outputs[0].text
        time_elapsed = getattr(output, 'time_elapsed', None)  # may not be available in all versions
        if time_elapsed is not None:
            result = f"[Prompt]: {prompt}\n[Generated]: {generated_text}\n[Time]: {time_elapsed:.2f}s\n{'-'*50}\n"
        else:
            result = f"[Prompt]: {prompt}\n[Generated]: {generated_text}\n{'-'*50}\n"
        results.append(result)
        print(result)

    # 6. Save results to a text file
    with open("vllm_batch_inference_results.txt", "w", encoding="utf-8") as f:
        f.write("".join(results))

    print("Batch inference completed. Results saved to vllm_batch_inference_results.txt")
```

### 2.2.4 核心参数详细说明

离线批量推理的核心参数分为「LLM 类参数」（模型加载相关）和「SamplingParams 类参数」（推理生成相关），以下为官方详细说明，明确参数作用、默认值、适用场景，避免配置错误。

#### 1. LLM 类核心参数（模型加载相关）

|参数名称|官方默认值|官方说明及适用场景|
|---|---|---|
|model|无（必填）|Hugging Face 模型名称（如 Qwen/Qwen2.5-1.5B-Instruct）或本地模型路径，离线推理的核心参数，必须填写。|
|trust_remote_code|False|是否信任远程代码，适配自定义模型（如 Qwen、LLaMA），官方推荐设为 True，否则可能无法加载模型。|
|dtype|auto|推理精度：auto 自动适配模型精度；float16 内存占用低，适合显存紧张场景；bfloat16 精度更高，适合显存充足场景。|
|max_model_len|2048|模型最大序列长度，需大于 prompt 长度+max_tokens，官方推荐设为 4096，适配大多数场景。|
|load_format|auto|模型加载格式：auto 自动选择最优方式；pytorch 加载 PyTorch 模型；safetensors 加载安全张量模型（推荐）。|
|                      |            |                                                              |
|tensor_parallel_size|1|张量并行 GPU 数量，单节点多 GPU 场景需设为 GPU 数量，用于多 GPU 批量推理（参考第五章并行技术）。|
#### 2. SamplingParams 类核心参数（推理生成相关）

|参数名称|官方默认值|官方说明及适用场景|
|---|---|---|
|temperature|0.7|控制推理结果的随机性：0 为确定性输出（无变化），1 为最大随机性，0.7 为官方推荐，兼顾多样性和准确性。|
|max_tokens|16|最大生成 token 数，避免生成过长内容，批量推理推荐设为 100~500，根据实际需求调整。|
|top_p|0.95|核采样，控制生成内容的多样性：值越小，生成内容越集中；值越大，多样性越强，官方默认 0.95。|
|stop|None|停止符列表，遇到列表中的字符立即停止生成，适配指令模型（如 Qwen），推荐设置 ["\n", "###"]。|
|presence_penalty|0.0|重复内容惩罚，减少相同内容的重复生成，批量推理推荐设为 0.1~0.2，避免生成冗余内容。|
|frequency_penalty|0.0|频率惩罚，降低高频词汇的出现概率，优化生成内容的丰富度，批量推理推荐设为 0.1~0.2。|
### 2.2.5 扩展：单条 prompt 推理简化写法

若无需批量推理，仅需单条 prompt 测试，官方提供简化写法，无需准备批量 prompt 列表，代码更简洁，适合快速验证模型是否正常加载、推理是否正常，同时体现 3 个关键类的简化使用。

```python
from vllm import LLM, SamplingParams

# 1. 配置采样参数（SamplingParams 类实例，与批量推理一致）
sampling_params = SamplingParams(temperature=0.7, max_tokens=100, top_p=0.95)

# 2. 初始化 LLM 引擎（LLM 类实例，与批量推理一致）
llm = LLM(model="Qwen/Qwen2.5-1.5B-Instruct", trust_remote_code=True, dtype="auto")

# 3. 单条 prompt 推理（调用 generate() 方法，传入单条字符串，返回 LLMOutput 列表）
prompt = "解释 VLLM 单条 prompt 推理的简化写法优势"
outputs = llm.generate(prompt, sampling_params)  # outputs 长度为 1

# 4. 解析结果（提取 LLMOutput 实例的核心属性）
generated_text = outputs[0].outputs[0].text  # 第一条 LLMOutput 实例的生成结果
time_elapsed = outputs[0].time_elapsed
print(f"【输入】：{prompt}\n【输出】：{generated_text}\n【推理耗时】：{time_elapsed:.2f}s")

```

官方说明：单条推理与批量推理的核心逻辑、关键类使用完全一致，仅输入 prompt 类型不同（单字符串 vs 字符串列表），可根据实际需求选择写法。

### 2.2.6 注意事项

以下注意事项均来自 VLLM 官方文档，结合 3 个关键类的使用，是离线批量推理避坑关键，必须严格遵守，否则可能导致推理失败、显存溢出、结果异常等问题。

- 模型适配：确保使用的模型支持 VLLM 推理（官方支持绝大多数 Hugging Face 指令模型，如 Qwen、LLaMA、Mistral 等），非指令模型需调整 prompt 格式，否则 LLM 类加载模型可能失败。

- 显存充足：LLM 类初始化时会加载模型到 GPU，批量推理前需确认 GPU 显存大于模型大小（含 KV 缓存），1.5B 模型（float16）约占用 3~4GB 显存，7B 模型约占用 13~15GB 显存，批量越大，KV 缓存占用越高。

- 参数匹配：max_model_len（LLM 类参数）必须大于「prompt 长度 + max_tokens（SamplingParams 类参数）」，否则会报序列长度溢出错误，建议预留一定冗余。

- 批量大小：根据 GPU 显存调整批量 prompt 数量，显存充足时可增加批量大小（如 100 条/批），显存紧张时减少批量大小（如 10 条/批），避免 OOM 错误，减少 LLM 类的资源占用压力。

- 结果解析：outputs 是 LLMOutput 实例列表，每个元素对应一条 prompt 的推理结果，需通过 output.outputs[0].text 获取生成内容，不可直接打印 outputs，避免解析失败。

- 模型加载：若模型为本地路径，需确保路径正确，且模型文件完整（无缺失权重文件），否则 LLM 类初始化会报模型加载失败错误。

- 版本兼容：确保 VLLM 版本与模型版本兼容.

# 第8章 多模态输入

vLLM 通过 vLLM-Omni 框架实现多模态输入推理，打破单一文本输入的限制，支持图像、音频等多类型输入，兼容主流多模态模型，适配离线推理与在线 API 服务两大场景。本章严格遵循官方规范，从框架介绍、模型支持、数据传递、请求构造四个维度，系统讲解 vLLM 多模态输入的核心逻辑与实操落地，兼顾教学易懂性与生产实用性。

## 8.1 vLLM-Omni 框架介绍

vLLM-Omni 是 vLLM v0.7.0 官方推出的多模态推理框架，核心定位是「统一多模态输入接口、高效适配多模态模型推理」，基于 vLLM 核心推理引擎优化，兼顾推理性能与兼容性，无需额外开发即可实现多模态输入的高效推理。

### 8.1.1 核心架构

vLLM-Omni 框架采用「分层架构」设计，分为三层，职责清晰，适配教学讲解：

1. **输入解析层**：负责解析不同类型的多模态输入（图像、音频），将其转换为模型可识别的张量格式，支持 URL、本地文件、PIL 图像、音频数组等多种输入形式，自动完成格式归一化。
   
2. **模型适配层**：对接各类主流多模态模型（LLaVA、Qwen-VL 等），统一模型调用接口，屏蔽不同模型的推理差异，无需修改代码即可切换不同多模态模型。
   
3. **推理引擎层**：复用 vLLM 核心推理引擎（基于 PagedAttention 优化），实现多模态推理的高效调度，兼顾高吞吐与低延迟，支持批量推理与并发请求。
   

### 8.1.2 核心优势

- 兼容性强：统一适配主流多模态模型，无需额外适配代码，降低多模态推理的开发成本。

- 性能优异：复用 vLLM 核心推理优化，多模态推理速度比原生框架快 3~5 倍，支持高并发请求。

- 接口统一：提供标准化的多模态输入接口，离线推理与 API 服务复用同一套核心逻辑，降低学习与运维成本。

- 轻量集成：vLLM v0.7.0 已内置 vLLM-Omni 框架，无需额外安装，直接通过参数启用即可。

## 8.2 支持的多模态模型

vLLM-Omni 框架适配 vLLM v0.7.0 版本主流多模态模型，涵盖图像理解、语音转文字等核心场景，官方明确支持的模型如下，按场景分类梳理，便于教学记忆：

### 8.2.1 图像理解类模型（文本+图像输入）

- **LLaVA（Large Language and Vision Assistant）**：基于 LLaMA 模型扩展的多模态模型，支持图像描述、图像问答、图像推理等任务，适配中小规模图像理解场景。
      

- **Qwen-VL（通义千问视觉版）**：阿里推出的多模态模型，支持中英文双语，具备更强的图像细节理解能力，适配多语言图像问答场景。
      

- **InternVL**：商汤科技推出的开源多模态模型，支持超高清图像理解、多图像对比等任务，适配高精度图像推理场景。
      

### 8.2.2 语音转文字类模型（音频+文本输入）

- **Whisper**：OpenAI 推出的语音转文字模型，支持多语言语音识别、语音翻译，vLLM-Omni 优化其推理速度，适配实时语音转文字场景。
      

### 8.2.3 模型部署注意事项（教学重点）

- 模型下载：需从 Hugging Face Hub 下载对应模型的完整权重（含视觉/音频编码器），确保模型文件完整。

- 依赖安装：部分模型需额外安装依赖（如 Whisper 需安装 ffmpeg，Qwen-VL 需安装 sentencepiece），官方推荐命令：`pip install ffmpeg sentencepiece pillow`。

- 版本兼容：确保模型版本与 vLLM v0.7.0 兼容，避免因模型版本过高/过低导致部署失败（官方推荐使用模型最新稳定版）。

## 8.3 多模态数据传递方式

vLLM-Omni 支持四种核心多模态数据传递方式，适配不同使用场景（本地测试、远程调用、代码开发），所有传递方式均已标准化，无需额外格式转换，以下为官方详细说明与实操示例：

### 8.3.1 URL 传递（远程资源）

适用于远程图像/音频资源，直接传入资源 URL，vLLM-Omni 自动下载、解析并转换为模型可识别格式，无需本地存储资源。

实操示例（图像 URL）：

```python
#为了实验方便先下载模型 在下载之前需要先执行登陆###要使用vllm支持的模型架构
huggingface-cli download llava-hf/llava-1.5-7b-hf \
  --local-dir ./LLaVA-7B-v1.5 \
  --local-dir-use-symlinks False
(base) root@ac14c8d0800b:/data# cat  vllm-ommi.py

from vllm import LLM, SamplingParams  # 导入 vLLM 相关的类和函数
from PIL import Image                  # 导入 PIL 用于处理图像
import requests                        # 导入 requests 用于下载图像
from io import BytesIO                  # 导入 BytesIO 用于处理二进制数据

def main():
    # 1. 加载模型（使用本地已下载的 LLaVA-1.5 模型）
    model_path = "./LLaVA-7B-v1.5"                     # 本地模型路径
    llm = LLM(
        model=model_path,                               # 指定模型路径
        trust_remote_code=True,                         # 允许执行远程代码（LLaVA 需要）
        max_model_len=2048,                              # 根据 GPU 显存调整最大序列长度
        gpu_memory_utilization=0.9,                      # 使用 90% 的 GPU 显存
    )

    # 2. 配置采样参数
    sampling_params = SamplingParams(
        temperature=0.2,          # 温度参数，控制生成文本的随机性
        max_tokens=100,           # 最大生成的 token 数量
        top_p=0.95,               # 核采样概率，保留累积概率达到 0.95 的 token
    )

    # 3. 从 URL 加载一张示例图片
    url = "http://images.cocodataset.org/val2017/000000039769.jpg"  # COCO 数据集图片（两只猫）
    response = requests.get(url)                     # 发送 HTTP 请求获取图片
    image = Image.open(BytesIO(response.content))    # 将二进制数据转换为 PIL Image 对象

    # 4. 准备提示词（LLaVA 模型需要特定的对话格式）
    prompt = "USER: <image>\nDescribe this image in detail.\nASSISTANT:"

    # 5. 执行推理，传入图像和提示词
    outputs = llm.generate(
        {
            "prompt": prompt,                          # 文本提示
            "multi_modal_data": {"image": image},      # 多模态数据，此处为图像
        },
        sampling_params=sampling_params,                # 采样参数
    )

    # 6. 打印生成的文本
    generated_text = outputs[0].outputs[0].text        # 提取第一个输出的文本
    print("Generated:", generated_text)                 # 输出结果

if __name__ == "__main__":
    main()  # 确保在主进程中执行，避免多进程启动问题
    
从代码中可以看出这是多模态推理的关键点有以下几个：
1. 模型类型：使用了 LLaVA 模型（model_path = "./LLaVA-7B-v1.5"），LLaVA 是一个典型的多模态（视觉-语言）模型，能同时处理图像和文本。
2. 提示词中的图像占位符：prompt = "USER: <image>\nDescribe this image in detail.\nASSISTANT:" 中的 <image> 标记表示此处需要插入图像特征，这是多模态模型常见的输入格式。
3. multi_modal_data 参数：在调用 llm.generate() 时，通过 multi_modal_data={"image": image} 显式地将图像数据（PIL.Image 对象）传递给模型，这是 vLLM 支持多模态输入的标准方式。
```

### 8.3.4 音频数组传递（音频开发场景）

适用于音频信号处理场景，直接传入音频数组（如 numpy 数组），支持自定义采样率、声道数，适配专业音频处理需求。

实操示例（音频数组）：

```python
(base) root@ac14c8d0800b:/data# cat vllm-ommi-music.py 
from vllm import LLM, SamplingParams
import librosa
import soundfile as sf
from pathlib import Path

def main():
    # 1. Configure model and parameters
    model_path = "Qwen/Qwen2-Audio-7B-Instruct"  # or local path like "./qwen2-audio-7b"
    
    # For audio models, it is recommended to install vllm[audio] extension and set limit_mm_per_prompt
    llm = LLM(
        model=model_path,
        trust_remote_code=True,
        max_model_len=4096,               # Adjust based on GPU memory
        max_num_seqs=5,                    # Limit parallel sequences
        limit_mm_per_prompt={"audio": 2},  # Maximum number of audios allowed per prompt
        gpu_memory_utilization=0.9,
    )

    # 2. Sampling parameters
    sampling_params = SamplingParams(
        temperature=0.2,
        max_tokens=128,
        top_p=0.95,
        stop_token_ids=None,
    )

    # 3. Prepare local audio files
    #    Supported formats: wav, mp3, flac, m4a, etc. (requires librosa support)
    audio_files = [
        "example1.wav",  # Replace with actual file path
        "example2.mp3",  # Can pass multiple audios (needs limit_mm_per_prompt setting)
    ]
    
    # Load audio using librosa (returns numpy array and sample rate)
    audios = []
    for audio_file in audio_files:
        if Path(audio_file).exists():
            waveform, sample_rate = librosa.load(audio_file, sr=None)  # Keep original sample rate
            # Note: The model may require a specific sample rate (e.g., 16000), adjust according to processor
            audios.append((waveform, sample_rate))
        else:
            print(f"Warning: {audio_file} not found, skipping")
    
    if not audios:
        print("No valid audio files found")
        return

    # 4. Build prompt (Qwen2-Audio uses ChatML format and special placeholders)
    #    Single audio scenario
    if len(audios) == 1:
        prompt = (
            "<|im_start|>system\nYou are a helpful assistant.<|im_end|>\n"
            "<|im_start|>user\n"
            "Audio 1: <|audio_bos|><|AUDIO|><|audio_eos|>\n"
            "Please describe the content of this audio in detail.<|im_end|>\n"
            "<|im_start|>assistant\n"
        )
    #    Multi-audio scenario (must match limit_mm_per_prompt)
    else:
        audio_placeholders = "\n".join([
            f"Audio {i+1}: <|audio_bos|><|AUDIO|><|audio_eos|>"
            for i in range(len(audios))
        ])
        prompt = (
            "<|im_start|>system\nYou are a helpful assistant.<|im_end|>\n"
            "<|im_start|>user\n"
            f"{audio_placeholders}\n"
            "What are the contents of these audios? Please describe each one.<|im_end|>\n"
            "<|im_start|>assistant\n"
        )

    # 5. Build multimodal input
    #    multi_modal_data format is {"audio": [(waveform, sample_rate), ...]}
    inputs = {
        "prompt": prompt,
        "multi_modal_data": {"audio": audios}
    }

    # 6. Run inference
    outputs = llm.generate(inputs, sampling_params=sampling_params)

    # 7. Output results
    for i, output in enumerate(outputs):
        generated_text = output.outputs[0].text
        print(f"\n=== Result for audio group {i+1} ===")
        print(generated_text)

if __name__ == "__main__":
    main()

```

## 8.4 离线推理中的多模态请求构造

离线推理（本地批量推理）是多模态场景的常用方式，核心是构造标准化的多模态请求，支持批量处理图像、音频数据，vLLM-Omni 提供简洁的请求构造方式，适配批量推理需求，以下为官方推荐的实操流程：

### 8.4.1 核心构造逻辑

离线多模态请求构造的核心的是「多模态输入对象+文本 prompt 拼接」，步骤如下：

1. 导入 vLLM 核心模块及多模态相关依赖（如 PIL 用于图像读取）；
2. 初始化 LLM 引擎，加载本地多模态模型（如 LLaVA-1.5），配置显存占用、最大序列长度等核心参数；
3. 配置采样参数，控制生成文本的准确性、多样性与长度；
4. 加载本地多模态数据（如图像），构造标准化输入；
5. 按模型要求拼接 prompt（如 LLaVA 专属的 USER/ASSISTANT 格式），关联多模态输入数据；
6. 调用推理方法，执行离线推理，解析并输出结果。

### 8.4.2 批量离线推理实操示例（多图像批量处理）


### 8.4.3 离线推理注意事项（官方强调）

- 批量大小调整：通过 `--max-batch-size` 参数调整批量大小，根据 GPU 显存调整（显存充足可增大，避免 OOM）。

- 数据格式校验：确保所有多模态数据格式统一（如图像尺寸、音频采样率），避免因格式不一致导致推理失败。

- 日志输出：通过 `--verbose` 参数开启详细日志，便于排查推理过程中的问题（如数据解析失败、模型加载异常）。

## 8.5 API 服务中的多模态请求：兼容 OpenAI Vision/Audio 格式

vLLM 多模态 API 服务完全兼容 OpenAI Vision/Audio 接口格式，无需修改客户端代码，即可直接替换 OpenAI 服务，降低迁移成本，适配在线多模态服务场景，以下为官方推荐的部署与请求实操：

### 8.5.1 启动多模态 API 服务（官方命令）

启动 vLLM 服务时，指定多模态模型，自动启用 vLLM-Omni 框架，兼容 OpenAI 接口格式：

```bash
# 示例1：启动 LLaVA 模型，支持图像输入 API（兼容 OpenAI Vision）
(base) root@ac14c8d0800b:/data# vllm serve llava-hf/llava-1.5-7b-hf \ 
                      					--trust-remote-code   \
                                --limit-mm-per-prompt '{"image": 1}'   \
                                --max-model-len 2048   \
                                --gpu-memory-utilization 0.9  \
                                --port 8000

# 示例2：启动 wen/Qwen3-Omni-30B-A3B-Instruct 模型，支持音频输入 API（兼容 OpenAI Audio）
vllm serve Qwen/Qwen3-Omni-30B-A3B-Instruct \
    --trust-remote-code \
    --limit-mm-per-prompt '{"audio": 2, "image": 2}' \
    --dtype bfloat16 \
    --max-model-len 32768 \
    --port 8000
```

### 8.5.2 图像 API 请求（兼容 OpenAI Vision 格式）

使用 curl 发送请求，完全兼容 OpenAI Vision 接口参数，支持 URL/Base64 图像传递：

```bash
# 示例：发送图像 URL 请求，询问图像内容

      ]
    }

  "temperature": 0.7,

}'

```

### 8.5.3 音频 API 请求（兼容 OpenAI Audio 格式）

支持语音转文字（transcriptions）接口，兼容 OpenAI Audio 格式，可直接复用 OpenAI 客户端代码：

```bash

-H "Authorization: Bearer your-api-key" \
-F "file=@./test-audio.wav" \

```

### 8.5.4 API 服务注意事项（教学重点）

- 接口兼容：所有请求参数、响应格式与 OpenAI 完全一致，可直接替换 OpenAI API 地址使用。

- 并发控制：通过 `--max-num-batched-tokens` 参数控制并发请求数量，避免 GPU 过载。

- Base64 支持：图像/音频可通过 Base64 编码传递（适合无网络访问的场景），只需将 image_url.url 替换为 Base64 编码字符串。

- 跨域支持：生产场景可添加 `--cors-allow-origins "*"` 参数，支持跨域请求，适配前端调用。

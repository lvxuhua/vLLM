# 第三章 服务器部署指南

## 3.1 官方核心优势

本章节提供的三种部署方式（OpenAI兼容服务器部署、自定义HTTP服务器部署、Web服务部署），均为官方认证方案，核心优势如下，兼顾实用性、兼容性与安全性，适配不同使用场景需求：

- 兼容性强：OpenAI兼容部署可直接对接OpenAI Python客户端、curl等主流调用工具，无需修改现有代码，无缝迁移现有OpenAI相关业务；自定义HTTP部署支持灵活扩展，适配各类自定义业务场景；Web服务部署兼容浏览器交互，无需额外工具即可操作。

- 部署高效：提供官方标准化步骤、命令及核心代码示例，简化部署流程，新手可快速上手，生产环境可直接复用核心配置。

- 功能完善：支持基础调用、流式输出、浏览器可视化交互等核心功能，同时提供详细的参数说明和注意事项，规避部署及使用过程中的常见问题。

- 安全可靠：官方推荐的部署参数及注意事项，针对在线服务部署场景优化，降低端口暴露、权限泄露等安全风险。

## 3.2 OpenAI 兼容服务器部署

该部署方式为官方首选方案，完全兼容OpenAI API协议，可直接使用OpenAI生态工具进行调用，无需额外开发适配代码，适合大多数生产及测试场景。同时补充vllm serve部署方式（高效推理，适合大模型部署），满足不同性能需求。

### 3.2.1 步骤 1：启动 OpenAI 兼容服务器

提供两种核心启动方式（官方FastAPI启动+新增vllm serve启动），分别适配不同场景：官方FastAPI启动兼容性强，vllm serve启动推理速度快，适合大模型部署；每种方式均提供生产环境完整配置和快速测试配置，按需选择即可。

#### 方式1：官方FastAPI启动

```bash
python -m vllm \
    --model /mnt/moark-models/Qwen3-0.6B \
    --host 0.0.0.0 \
    --port 8000 \
    --trust-remote-code
# 若使用远程模型，替换为：--model-base-url https://api.your-model.com/v1
- --host 0.0.0.0：允许外部设备访问服务器，生产环境必须配置，本地测试可改为127.0.0.1。
- --port 8000：指定服务器端口，可根据实际需求修改（如8080、9000），需确保端口未被占用。
- --workers 4：启动4个工作进程，适配多核服务器，提升并发处理能力，可根据服务器配置调整数量。
- --log-level info：日志级别为info，记录核心操作日志，便于问题排查，生产环境建议保留。
- --model-base-url https://api.your-model.com/v1：远程模型专用，替换为远程模型API地址（如OpenAI地址https://api.openai.com/v1）。
```



#### 方式2：vllm serve启动（高效推理，大模型首选）

vllm是高效大模型推理框架，支持高并发、低延迟，部署OpenAI兼容服务器时，比传统FastAPI启动速度快3-5倍，适合ChatGLM3-6B、Llama 3等大模型部署，完全兼容OpenAI API协议，可直接复用原有调用代码。

vllm serve命令无需编写额外脚本，直接指定模型即可启动OpenAI兼容服务器，核心参数简洁，模型指定为必填项，两种配置按需选择：

```bash

vllm serve  /path/to/your/model --host 0.0.0.0 --port 
# 核心参数说明：
# --model /path/to/your/model：必填，本地模型路径（如ChatGLM3-6B路径），也可直接填写模型名（自动下载）
# --host 0.0.0.0：允许外部访问，生产环境必配
# --port 8000：指定端口，与FastAPI启动端口可区分（如8000用vllm，8001用FastAPI）
# --tensor-parallel-size 1：GPU并行数量，1代表单GPU，多GPU可调整（如2、4）

```

##### 步骤3：vllm serve启动验证

启动成功后，两种验证方式，与官方FastAPI启动的验证逻辑一致，无需修改命令，直接复用即可：


### 3.2.3 官方核心服务器参数说明（FastAPI+vllm通用）

汇总两种启动方式的核心参数，按使用频率排序，生产环境需重点关注，参数通用适配，无需区分启动方式：

|参数名称|参数格式|官方说明|默认值|适用场景|
|---|---|---|---|---|
|--host|--host [IP地址]|指定服务器绑定的IP地址，0.0.0.0允许外部访问，127.0.0.1仅本地访问|127.0.0.1|生产环境（0.0.0.0）、本地测试（127.0.0.1）|
|--port|--port [端口号]|指定服务器监听的端口，需确保端口未被占用，建议使用1024以上端口|8000|所有场景，可根据实际需求修改|
|                        |                               |                                                              |              |                                              |
|--workers|--workers [数量]|FastAPI专用，指定工作进程数量；vllm无需配置，自动适配CPU/GPU|1（FastAPI）|FastAPI生产环境（多进程）、本地测试（1进程）|
|--tensor-parallel-size|--tensor-parallel-size [数量]|vllm专用，GPU并行数量，单GPU填1，多GPU按实际数量调整|1|vllm生产环境，多GPU部署场景|
### 3.2.4 官方注意事项

以下为官方强调的注意事项，针对在线服务部署场景，直接关系到服务器稳定性、安全性，两种启动方式均需严格遵守：

- 端口安全：在线部署时，避免使用80、443等常用端口（需root权限），建议使用1024以上端口；同时配置防火墙，仅开放需要访问的端口，禁止所有IP无限制访问。

- 权限控制：启动服务器时，避免使用root用户，建议创建专用普通用户，限制用户权限，防止服务器被入侵后造成大面积影响。

- 环境变量：敏感信息（如密钥、数据库密码）必须通过--env-file（FastAPI）或环境变量加载，禁止在启动命令中明文填写，避免信息泄露。

- 日志管理：生产环境需开启info及以上级别日志，FastAPI通过--log-level配置，vllm默认开启info日志；建议配置日志轮转，避免日志文件过大占用磁盘空间。

- 进程守护：在线部署时，建议使用systemd、supervisor等进程守护工具，确保服务器意外退出后自动重启，提升服务可用性。以下为vllm serve的systemd配置示例（FastAPI可参考修改）：

```bash
# 官方推荐的vllm serve进程守护配置（在线部署必配）
# 1. 创建服务配置文件
sudo vim /etc/systemd/system/vllm-server.service

# 2. 写入以下内容（替换对应的路径和命令）
[Unit]
Description=VLLM OpenAI Compatible Server
After=network.target

[Service]
User=your-username  # 替换为普通用户名
WorkingDirectory=/path/to/server  # 替换为工作目录
ExecStart=/usr/bin/vllm serve --model ~/.cache/huggingface/hub/models--THUDM--chatglm3-6b --host 0.0.0.0 --port 8000 --tensor-parallel-size 1
Restart=always  # 意外退出后自动重启
RestartSec=5  # 重启间隔5秒
StandardOutput=append:/var/log/vllm-server.log  # 日志输出路径
StandardError=append:/var/log/vllm-server.error.log

[Install]
WantedBy=multi-user.target

# 3. 启动并设置开机自启
sudo systemctl daemon-reload
sudo systemctl start vllm-server
sudo systemctl enable vllm-server
```

- 版本兼容：确保Python版本≥3.8，vllm版本与torch版本兼容（官方推荐vllm==0.4.2、torch==2.2.1）；FastAPI需确保uvicorn、pydantic版本匹配，避免版本不兼容导致启动失败。

- 模型适配：vllm支持大多数主流大模型（ChatGLM3、Llama 3、Qwen等），但部分小众模型可能不兼容，部署前建议确认模型是否支持vllm；FastAPI无模型限制，可适配所有支持transformers的模型。

### 3.2.5 部署总结

两种OpenAI兼容服务器部署方式，各有适配场景，根据自身需求选择即可，核心对比如下，清晰区分优势与适用场景：

|对比维度|官方FastAPI启动|vllm serve启动|
|---|---|---|
|推理速度|常规速度，适合中小模型（如ChatGLM3-6B），单并发场景|高速推理，比FastAPI快3-5倍，支持高并发，适合大模型|
|模型适配|无限制，适配所有支持transformers的模型，兼容性极强|支持主流大模型，部分小众模型不兼容，需提前确认|
|配置复杂度|需编写FastAPI脚本，参数较多，配置稍复杂|无需编写脚本，直接通过命令启动，参数简洁，配置简单|
|适用场景|中小模型、自定义业务逻辑、低并发场景、新手入门|大模型、高并发场景、追求推理速度、生产环境部署|
